## Installing Rapier from NPM
Rapier is available as the [@dimforge/rapier2d](https://www.npmjs.com/package/@dimforge/rapier2d) and
[@dimforge/rapier3d](https://www.npmjs.com/package/@dimforge/rapier3d) NPM packages. You may add the following to
your `package.json`:

<Tabs
  groupId="dim"
  defaultValue="2D"
  values={[
    {label: 'Example 2D', value: '2D'},
    {label: 'Example 3D', value: '3D'},
  ]}>
  <TabItem value="2D">

```json
  "dependencies": {
    "@dimforge/rapier2d": "^0.1.5",
  }
```

  </TabItem>
  <TabItem value="3D">

```json
  "dependencies": {
    "@dimforge/rapier3d": "^0.1.5",
  }
```

  </TabItem>
</Tabs>

Because Rapier is actually a WebAssembly module, it has to be loaded asynchronously:

<Tabs
  groupId="dim"
  defaultValue="2D"
  values={[
    {label: 'Example 2D', value: '2D'},
    {label: 'Example 3D', value: '3D'},
  ]}>
  <TabItem value="2D">

```js
import('@dimforge/rapier2d').then(RAPIER => {
  // Use the RAPIER module here.
  let world = new RAPIER.World(0.0, -9.81);
})
```

  </TabItem>
  <TabItem value="3D">

```js
import('@dimforge/rapier3d').then(RAPIER => {
  // Use the RAPIER module here.
  let world = new RAPIER.World(0.0, -9.81, 0.0);
})
```

  </TabItem>
</Tabs>

See the `testbed3d/src/demos` and `testbed2d/src/demos` folders for examples on how to initialize a Rapier physics
world using these bindings.
